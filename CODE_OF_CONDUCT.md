# Code of Conduct

OpenCode Pilot is a small open-source project. The goal is simple: make it useful, safe, and pleasant to work on.

Everyone participating in this project — issues, pull requests, discussions, reviews, or any other community space — is expected to follow this Code of Conduct.

## Expected behavior

- Be respectful and constructive.
- Assume good intent first, but accept correction quickly.
- Keep feedback focused on the work, not the person.
- Give enough context when reporting a problem.
- Respect maintainer time and project scope.
- Prefer clear disagreement over vague negativity.

## Unacceptable behavior

- Harassment, threats, insults, or personal attacks.
- Racist, sexist, homophobic, transphobic, ableist, or otherwise discriminatory language.
- Sexualized language or unwanted attention.
- Repeatedly pushing the same request after a maintainer has made a decision.
- Opening hostile, spammy, or intentionally disruptive issues/PRs.
- Publishing private security details before a fix is available.

## Maintainer authority

The maintainer may edit, hide, lock, or delete comments; close issues or pull requests; block accounts; or take any other action needed to keep the project healthy.

This is not about gatekeeping. It is about keeping the project maintainable and safe for users.

## Reporting a problem

If someone violates this Code of Conduct, contact the maintainer privately. If the problem is a security issue, follow [`SECURITY.md`](SECURITY.md) instead of opening a public issue.

When reporting behavior, include:

- A link to the issue, PR, comment, or discussion.
- What happened.
- Why it violates this Code of Conduct.
- Any context that helps the maintainer understand the situation.

## Scope

This Code of Conduct applies to project spaces and public interactions related to OpenCode Pilot. It also applies when someone represents the project in another community.
